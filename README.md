# AIY Voice Kit Drivers (Kernel 6.x)

This repository contains out-of-tree Linux kernel modules and overlays for the **Google AIY Voice Bonnet v2** stack, updated for modern Raspberry Pi kernels (including 6.x series).

## Important Hardware Compatibility

This repo is for **Voice Bonnet v2** hardware (RT5645 codec + AIY MCU + KTD202x LED driver).

If your HAT EEPROM reports:
- `Google AIY audioHAT Board v1`

then you have **Voice HAT v1**, not Voice Bonnet v2. In that case, use the Raspberry Pi Voice HAT overlay/driver stack (for example `googlevoicehat-soundcard`) instead of these v2 modules.

## Repository Origin

- Upstream fork source: `https://github.com/kobayutapon/aiy-voicekit-drivers`
- Original AIY driver lineage comes from Google AIY DKMS packages (`aiyprojects-raspbian`).

## What Is Included

- `aiy/`
  - `aiy-io-i2c` (MCU / MFD)
  - `gpio-aiy-io`
  - `pwm-aiy-io`
  - `aiy-adc`
- `sound/`
  - `snd-aiy-voicebonnet`
  - `rt5645`
  - `rl6231`
- `leds/`
  - `leds-ktd202x`
- `overlays/`
  - Device tree overlay sources and packaging scripts

## Prerequisites

1. Raspberry Pi OS / Debian with matching kernel headers installed.
2. Build tools installed (`make`, `gcc`, etc.).
3. Running kernel headers available at:
   - `/lib/modules/$(uname -r)/build`

## Top-Level Build Commands

From repo root:

```bash
make
```

Builds all module groups:
- `aiy`
- `sound`
- `leds`

### Clean (new)

```bash
make clean
```

This runs kernel-module clean in all component directories and removes build artifacts generated by previous builds.

### Install

```bash
sudo make install
```

Installs all built `.ko` files to:
- `/lib/modules/$(uname -r)/extra`

Then runs:
- `depmod -a $(uname -r)`

## Component-Only Commands

You can still build individual sections:

```bash
make aiy
make sound
make leds
```

## Load and Verify

After install:

```bash
sudo modprobe aiy-io-i2c
sudo modprobe gpio-aiy-io
sudo modprobe pwm-aiy-io
sudo modprobe aiy-adc
sudo modprobe leds-ktd202x
sudo modprobe snd-aiy-voicebonnet
```

Check loaded modules:

```bash
lsmod | grep -Ei 'aiy|voicebonnet|rt5645|rl6231|ktd202x'
```

## Overlay Notes

For Voice Bonnet v2, the expected overlay is typically:

```ini
dtoverlay=aiy-voicebonnet
```

on Pi systems where that overlay is available/installed.

## Known Pitfalls

1. **Wrong board type**
   - Voice HAT v1 and Voice Bonnet v2 are different hardware families.
2. **Kernel headers mismatch**
   - Modules must be built against the exact running kernel.
3. **User-space permissions**
   - For ALSA access without sudo, user must be in `audio` group.
